# Endfield Balance Lab v58

정적 HTML/CSS/JavaScript로 구성된 캐릭터 전투 구조·용어 사전·파티 분석·분석 결과·제약 추천 사이트입니다.

## 페이지

- `index.html`: 캐릭터 전투 구조 데이터베이스
- `glossary.html`: 게임 용어 사전
- `party.html`: 캐릭터 4명 편성 및 파티 메커니즘 분석
- `results.html`: 내보낸 파티 분석 목록과 상세 분석 재현
- `constraints.html`: 저장된 분석 자료를 근거로 한 제약 추천

## v55 변경 사항

- 분석 결과 목록의 각 파티 정보 바를 `분석 펼치기` / `분석 접기` 버튼으로 개별적으로 열고 닫을 수 있습니다.
- 분석 결과 페이지의 상세 출력은 파티 분석 페이지와 같은 분석 클래스와 구성 양식을 사용합니다.
- 분석 결과 페이지 안내 문구를 `파티 분석 결과를 리스트로 제공합니다.`로 변경했습니다.
- 제약 추천은 특정 파티 하나를 선택하는 방식이 아니라, 저장된 모든 파티 분석을 종합해 추천합니다. 저장된 분석이 하나라면 해당 파티만을 기준으로 합니다.
- 제약 추천 근거에는 전체 저장 파티 수와 관련 경향이 확인된 파티 수를 함께 표시합니다.
- 제약 카드의 `[제약 기능]`, `[설명]`, `[추천 근거]`, `[예상 영향]` 글자 크기를 확대했습니다.

## 분석 JSON을 사이트에 고정하는 방법

현재 `분석 내보내기`는 JSON 다운로드와 브라우저 로컬 저장을 함께 수행합니다. GitHub Pages에 분석을 고정하려면 보관한 JSON들을 `saved-analyses.js`의 배열에 합쳐 배포하면 됩니다.

## 실행

웹 서버 또는 GitHub Pages에 폴더 전체를 배포하세요. 로컬에서는 각 HTML 파일을 직접 열어도 대부분 동작하지만, 브라우저 보안 정책에 따라 웹 서버 실행을 권장합니다.


## v56
- 파티 분석 페이지에서 분석 결과 페이지로 이동할 때 브라우저별 로컬 파일 확대 비율 차이를 감지해, 파티 분석 화면의 시각적 배율을 분석 결과 페이지에 맞춰 적용합니다.

- v57: 분석 결과를 펼쳐도 문서 전체 폭과 화면 비율이 변하지 않도록 상세 영역의 최소 너비를 격리했습니다.
- 제약 추천 종합 카드에서는 캐릭터 초상화를 최대 6명까지만 표시하고 나머지는 `+N`으로 묶습니다.


## v58
- 분석 결과 페이지에도 파티 분석 페이지와 동일한 우측 `캐릭터 빠른 조회` 패널을 추가했습니다.
- 오른쪽 화살표로 패널을 열고 닫을 수 있으며, 열린 너비만큼 분석 결과 본문이 옆으로 이동합니다.
- 캐릭터 이름은 첫 글자부터 검색하며, 선택한 캐릭터의 전투 스킬과 재능을 같은 패널 디자인으로 표시합니다.
- 분석 결과를 펼치거나 접어도 빠른 조회 패널의 열림 상태와 검색 결과가 유지됩니다.

## GitHub Pages 배포용 로컬 이미지 자산

이 배포본은 `EF_CHAR.zip`의 캐릭터 초상화, 속성 아이콘, 역할군 아이콘을 사이트 내부 `assets/` 폴더에 포함합니다.
외부 이미지 서버를 호출하지 않으며, GitHub Pages와 로컬 파일 실행에서 동일한 상대 경로를 사용합니다.

- 캐릭터 초상화: `assets/characters/<character-id>/portrait.png`
- 속성 아이콘: `assets/icons/elements/`
- 역할군 아이콘: `assets/icons/roles/`
- 자산 매핑: `skport-assets.js`
- 자산 목록: `assets/asset-manifest.json`

GitHub 저장소에는 이 폴더의 **내용 전체**를 최상단에 업로드하세요. `index.html`이 저장소 루트에 있어야 합니다.
